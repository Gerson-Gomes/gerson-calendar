# Initial Concept

A local-first, offline desktop calendar application built with Wails, Go, React, and SQLite.

# Product Guide

## Overview
Gerson Calendar is a privacy-focused, local-first calendar application designed specifically for the Linux desktop. It aims to modernize an existing calendar system with a modern React UI while providing a fast, lightweight, and completely offline experience.

## Target Audience
- **Linux Desktop Users**: Enthusiasts seeking a native, fast, and simple calendar tool.
- **Productivity Seekers**: Individuals who need a lightweight tool to manage events and attachments without cloud overhead.

## Goals
- **Modernization**: Replace or enhance an existing calendar system with a modern, responsive React-based interface.
- **Privacy First**: Ensure all data stays local with no cloud synchronization or tracking.
- **Performance**: Maintain a small footprint with fast startup times and low memory usage.

## Key Features
- **Local Storage (SQLite)**: Robust and reliable local data management for all calendar events.
- **Desktop Integration**: Deep integration with the Linux desktop, including native notifications and file management.
- **Event Management**: Core calendar functionality to create, view, and manage events.
- **File Attachments**: Ability to attach PDFs and other documents to calendar events.

## Platform Strategy
- **Linux (Primary)**: Focus on deep optimization and integration with the Linux desktop environment.
